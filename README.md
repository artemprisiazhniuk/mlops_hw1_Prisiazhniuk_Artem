# Модели ML в продакшн. Домашнее задание 1

**Цель проекта:** построение минимального, но полноценного MLOps-контура, обеспечивающего
- воспроизводимость экспериментов;
- контроль версий данных и моделей;
- автоматизацию процесса обучения и оценки модели;
- документирование процесса через DVC и MLflow.

**Как запустить:**
- Убедитесь, что в системе установлены git, python, python-pip, dvc
- Склонируйте репозиторий с помощью команды `git clone <repo>` и войдите в него с помощью `cd <repo>`
- Установите зависимости, указанные в *requirements.txt* при помощи `pip install -r requirements.txt`
- Получите данные для обучения через DVC при помощи `dvc pull`
- Задайте нужные параметры в файле *params.yaml*
- Запустите пайплайн обучения при помощи `dvc repro`

**Краткое описание пайплайна:**
Пайплайн работает с датасетом Iris, разделяет данные на обучающую и тестовую выборки, проводит обучение 
модели RandomForest на обучающей выборке и сохраняет тестовые метрики и модель в MLflow.

**UI MLflow:**
Чтобы запустить UI MLflow необходимо находясь в корневой директории проекта запустить локальный сервер через `mlflow ui --backend-store-uri sqlite:///mlflow.db`. Чтобы увидеть UI нужно следовать инструкциям в консоли (открыть адрес 127.0.0.1:5000 в браузере). В эксперименте *IRIS RandomForest* будут отражены проведенные эксперименты.
